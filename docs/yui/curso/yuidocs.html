Components

  • It is a distribution unit, it does not correspond to a class of objects or any other formal JavaScript concept
  • A single component might have objects in the YAHOO.widget branch and in the YAHOO.util branch
  • Some elements are misplaced, the YUI team knows and they'll fix it, eventually

Classes

  • Ok, JavaScript has no classes but that is what the heading says
  • When you first reach the page, you have to pick one of these

Files

  • These show the source files before they are concatenated into the distribution units
  • They represent the original source files, not those that are distributed
  • Don't bother with them

Properties

  • Lists object properties and constants
  • Constants are not really such, JavaScript does not have constants
  • Constant names are in all uppercase and their names are usually preceded by the object name since they are also static to remind you that you have to name them in full
  • The index does not list the properties of the objects it inherits from, these are mentioned at the end of the section

Methods

  • Lists all methods and functions
  • Static functions are often preceded by the class name
  • The index does not list the methods and functions of the objects it inherits from, these are mentioned at the end of the section

Events

  • These are custom events fired by the object
  • They are listened to by using the subscribe or on method (they are aliases)
  • By default, the event listener is executed in the context of the object
  • The index does not list the events fired by the objects it inherits from, these are mentioned at the end of the section
someObject.on('someEvent', function(args) {
    // here, this points to someObject
});

Configuration attributes

  • Some components take extra arguments in the form of 'configuration attributes'
  • When available, they are listed at the end of the index
  • They are usually set via the last optional argument of the object constructor, as an object literal with the configuration attribute as the property name
  • Once the object is created, they can also be set via
    • the set or read via get methods inherited from the Element object which is augmented by AttributeProvider or any other directly augmented by AttributeProvider
    • or via the cfg property setProperty and getProperty methods for components in the container family and menu.
  • Changes made to those that have been augmented by AttributeProvider may have immediate effects
  • Those that have a cfg property need a call to render to have any effect
  • Eventually, those components that use cfg will move to the AttributeProvider mechanism

Source files

  • The ultimate resource, when everything else fails, this is what counts
  • Three flavours
    • xxx-debug.js These are the original ones. They are not minified and contain calls to YAHOO.log
    • xxx.js Normal files, they are derived from the previous with the calls to YAHOO.log stripped out
    • xxx-min.js Minified or compressed files from the previous ones, these are the final item fit for deployment
  • The name convention for files is important because the YUI Loader filter property depends on it
  • Aggregated files, such as utilities.js and reset-fonts-grids come only in minified version
  • The YUI Loader automatically loads the aggregated file when enough single components contained in it are requested

Readmes

  • Though admittedly they should be read with any release, at least you should read them when whatever worked in the previous version stops working in the new release
  • They are available on-line and also with each distribution in the same folder along with each component

Examples

  • Unfortunately, the source for most of the code out there in the wild (all tables are called myDataTable)
  • They are just examples!!!
  • They are self-contained, in order to show everything in a single file
  • They don't follow any good practices regarding code organization, deployment, modularization, encapsulation and such
  • They are good for testing and experimenting, not for final deployment
  • Some need to have a PHP interpreter active to provide them with data
