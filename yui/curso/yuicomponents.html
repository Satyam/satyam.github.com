YUI components

Reserved Namespaces

  • All YUI components occupy only a single global variable name: YAHOO
  • Some properties and methods hang right from YAHOO
    • YAHOO.log to log into the YUI Logger
    • YAHOO.namespace to create other custom namespaces
    • YAHOO.register
      • used by all components to announce their existence
      • should be used by custom components when meant to be loaded via the YUI Loader
    • YAHOO.VERSION, YAHOO.BUILD constants identifying the library version
    • deprecated methods relocated elsewhere
      • augment
      • extend

YAHOO.env

  • contains several YUI internal (though accessible) variables for the library to keep track of its workings
  • the most useful property is ua (User Agent) which identifies the browser (within those supported)

YAHOO.lang

  • Contains many functions that supplement or replace missing JavaScript functionality:
    • isXxxx static functions for type checking
    • Object manipulation
      • extend
      • augment aka augmentProto
      • augmentObject
    • Missing JavaScript functionality
      • trim trims whitespaces from ends of string
      • substitute replaces variables into template
      • merge merges two objects, provides a way to clone objects
      • Theoretically, these functions should have augmented the native JavaScript objects (String.prototype.trim, Object.prototype.merge)
      • In practice, since most people don't use hasOwnProperty when enumerating, doing so would have broken countless pages
    • Misplaced functionality (should be under util)
      • dump handy to dump variables if not in Firebug
      • later a more versatile replacement for window.setTimeout
      • JSON coding and decoding of JSON data

YAHOO.util

  • most utility objects are under this namespace (too long to list)
  • they are spread over many of the YUI files, some misplaced, in DataSource:
    • YAHOO.util.Number.format
    • YAHOO.util.Date.format
  • Many members are static (don't need to be instanced)
    • the two mentioned above
    • YAHOO.util.Connect.asyncRequest
    • YAHOO.util.Get.script
    • YAHOO.util.Dom.get
    • YAHOO.util.Event.on
    • many more...

YAHOO.widget

  • All UI-related objects
  • Most are object prototypes, since there can be more than one of each in a single page
  • A few singletons
    • Logger, there should be only one
    • MenuManager, EditorInfo, singletons meant to coordinate in between instances
  • At least one misplaced
    • DateMath in Calendar should be under util

YAHOO.tool

  • Development tools, these are not expected to be delivered to end users
    • Profiler
    • various Test objects

YAHOO.example

  • This namespace is free to use
  • It is used by all the examples available in the YUI site
  • It is not used otherwise

Class hierarchy

There is none

  • JavaScript is not based on rigid classes but on flexible objects and prototypes
  • This provides flexibility but does not enforce discipline
  • In other environments, there would be a YAHOO.widget abstract class
  • YUI is not consistent
