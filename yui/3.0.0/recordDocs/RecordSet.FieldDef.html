Stores definitions for the fields in the RecordSet

Properties

Reference to the parent of this definition or null if none

Reference to the top-most field definition or to itself if at the top

Configuration Attributes

_parent - private FieldDef

Provided as an internal-use setting to read the parent of this node even before the initializer method gets called. The parent is required to find out the root, which is used in setting some of the other attributes

children - FieldDefs

Definitions of fields group under this field
Default Value: null

initialValue - any

Initial value for a field when no value is provided.
Default Value: null

label - string

Label to be shown for this field (defaults to field name)
Default Value: null

name - string

Name of the field for this definition.
Default Value: null

parent - FieldDef

Reference to the parent of this definition

primaryKey - Boolean

Signals that the field is a primary key. Fields marked as being a primaryKey will be indexed and can be used to search for Records They will also be included in any serialized output even when not dirty
Default Value: false

root - FieldDef

Reference to the top-most definition containing this definition

server - {object}

Formatting and parsing functions for exchanging data with the server

server.formatter - function or null

Function to be used for formatting the field value when it is sent to the server
Default Value: null

server.parser - function or null

Function to be used for parsing the field when it is received from the server
Default Value: null

sortCompareFunction - function or null

Function to sort records by when sorted by this field. If none provided (the default) the sortCompareFunction attribute for the whole RecordSet will be used and finally the regular lexical comparison. The function will receive:
  • a: first record to compare
  • b: second record to compare
  • desc: Boolean indicating the sort should be in descending order
  • fieldName: Name of the field to sort by
    • Default Value: null.

    type - string

    DataType information. It should be a reference to any of the types stored in DataType.Types
    Default Value: null

    ui - {object}

    Formatting and parsing functions for exchanging data with the user

    ui.formatter - function or null

    Function to be used for formatting the field value when it is presented to the user
    Default Value: null

    ui.parser - function or null

    Function to be used for parsing the field when it is received from some user input component
    Default Value: null
